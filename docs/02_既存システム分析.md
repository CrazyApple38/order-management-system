# 既存Excelシステム分析

## 1. 概要

### 1.1 システム構成
現在のExcelシステムは2つの主要ファイルで構成：

| ファイル | 役割 |
|----------|------|
| **受注簿.xlsm** | 契約先・現場情報のマスタ、日毎の人数管理 |
| **業務管理計画書.xlsm** | 日毎の詳細な人員配置計画 |

### 1.2 コード規模
- **総VBAコード行数**: 約26,000行
- **受注簿側**: 約6,000〜7,000行
- **業務管理計画書側**: 約19,000〜20,000行

---

## 2. 受注簿のデータ構造

### 2.1 シートの列構造

| 列 | 項目 | 説明 |
|----|------|------|
| A | 本社/支店 | 組織区分 |
| B | 区分カテゴリ | 高速SA、高速、一般道など（ソート用） |
| C | 昼/夜 | シフト区分 |
| G | 契約先名 | 契約先の名称 |
| H | 業務名 | 具体的な業務内容 |
| I以降 | 日付列 | 各日の配置人数（セル値） |

### 2.2 セルコメント（メモ）の構造

日付セルにはコメント（メモ）で詳細情報を保持：

```
現場名: ○○現場
時間: 8:00-17:00
集合: △△駅 7:30
配置: 山田太郎、鈴木一郎、佐藤花子...
連絡: 担当者への連絡事項
備考: その他の情報
```

### 2.3 データ型定義（VBA）

```vba
' 受注簿から読み込んだデータ
Type OrderBookRowData
    ColA As String           ' 本社/支店
    ColB As String           ' 区分カテゴリ
    ColC As String           ' 昼/夜
    ColG As String           ' 契約先名
    ColH As String           ' 業務名
    CellValue As Variant     ' 人数
    SiteName As String       ' 現場名
    WorkTime As String       ' 時間
    MeetingPlace As String   ' 集合/出発
    Remarks As String        ' 備考
    Assignment As String     ' 配置（社員名リスト）
    Contact As String        ' 連絡
    TotalEmployees As Long   ' 総人数
End Type
```

## 3. 業務管理計画書のデータ構造

### 3.1 シートの列構造

| 列 | 定数 | 項目 | 説明 |
|----|------|------|------|
| A | COL_A=1 | 本社/支店 | 組織区分 |
| B | COL_B=2 | 契約先+業務名 | 改行区切りで結合 |
| C | COL_C=3 | 連絡 | 連絡事項 |
| D | COL_D=4 | 集合 | 集合場所・時間 |
| E | COL_E=5 | 時間 | 作業時間 |
| F | COL_F=6 | 人数 | 配置人数 |
| G〜P | 7〜16 | 配置1〜10 | 社員名（縦書き） |
| Q | COL_Q=17 | 備考 | 備考欄 |
| R〜W | 18〜23 | 配置11〜17 | 会社/休み行用 |

### 3.2 ブロック構造

データは3行1ブロックで管理：

```
ブロックタイプ:
- btSite (1)      : 現場ブロック
- btSiteExtra (2) : 現場追加ブロック（11人目以降）
- btCompany (3)   : 会社ブロック
- btYasumi1 (4)   : 休み（申請なし）
- btYasumi2 (5)   : 休み（申請あり）

定数:
- BLOCK_ROWS_PER_BLOCK = 3    # 1ブロック3行
- BLOCK_START_ROW = 3         # データ開始行
- MAX_ASSIGN_PER_ROW_SITE = 10    # 現場行1行の最大配置
- MAX_ASSIGN_PER_ROW_SPECIAL = 17 # 特殊行1行の最大配置
```

## 4. 主要機能

### 4.1 受注簿の機能

| 機能 | モジュール | 説明 |
|------|-----------|------|
| ソート | 受注簿ソート.bas | 会社→区分→契約先→業務名→昼/夜の順でソート |
| ソート設定UI | 受注簿ソート設定UI.bas | 優先順位のカスタマイズ |
| 土日祝色付け | 土日祝色付け.bas | 土日祝日のセル色変更 |
| 表示非表示 | 表示非表示.bas | 行の表示/非表示切替 |
| 勤務表印刷 | Print出勤表.bas | 勤務表の印刷機能 |

### 4.2 業務管理計画書の機能

| 機能 | モジュール | 説明 |
|------|-----------|------|
| データ取得 | GetOrderBookDataV2.bas | 受注簿からデータを取得・転記 |
| 差分検出 | DiffDataDetector.bas | 受注簿との差分を検出 |
| 差分反映 | DiffDataApplier.bas | 検出した差分を適用 |
| 行挿入 | InsertRows.bas | 新規データ用の行挿入 |
| 行圧縮 | CompactRows.bas | 空白行の削除 |
| データ削除 | ClearData.bas | データの一括削除 |
| 東央警備転記 | TohoTransferV2.bas | 東央警備専用の転記処理 |

### 4.3 データフロー

```
受注簿シート
    │
    │ OrderBookReader.ReadOrderBookData()
    │   ├─ 日付列を検索
    │   ├─ 各行のコメントを解析
    │   └─ OrderBookRowData配列に格納
    ↓
WorkPlanWriter.WriteOrderBookData()
    │   ├─ ブロック構造（3行単位）で配置
    │   ├─ 配置情報をG〜P列に配置
    │   ├─ 10人超過時は追加ブロック作成
    │   └─ 昼/夜で文字色変更（黒/赤）
    ↓
業務管理計画書シート
```

## 6. Webシステム化に向けた考慮事項

> **開発方針**: React + Node.js（Next.js）によるWebアプリケーションを先行開発し、その後Excel VBA版UIを後続開発する。以下の考慮事項は主にReact Webアプリ構築に適用される。

### 6.1 データモデルの変換

**Excelの構造 → RDB設計のポイント**

| Excel | データベース |
|-------|-------------|
| セルコメント | 正規化したカラムに分解 |
| 日付列（横方向） | 日付を行として正規化 |
| 3行ブロック | 単一レコードに統合 |
| ヘルパー列（ソート用） | ORDER BY句で対応 |

### 6.2 移行が必要なデータ

1. **マスタデータ**
   - 契約先マスタ
   - 現場マスタ
   - 社員マスタ
   - 区分マスタ（高速SA、高速、一般道など）

2. **トランザクションデータ**
   - 日毎の配置情報
   - 配置履歴

### 6.3 セルコメントの分解

```
セルコメント → 正規化されたカラム

現場名    → site_name
時間      → work_time_start, work_time_end
集合      → meeting_place, meeting_time
配置      → assignments テーブル（社員との関連）
連絡      → contact_notes
備考      → remarks
```

### 6.4 主要な画面要件

| 画面 | Excelでの対応 | React Webアプリでの実装 |
| ---- | ------------- | ----------------------- |
| 受注一覧 | 受注簿シート | データグリッド + フィルタ（Reactコンポーネント） |
| 日別配置 | 業務管理計画書シート | カレンダーグリッド（Reactコンポーネント） |
| ソート設定 | frmSortSettings | 設定ダイアログ（Reactモーダル） |
| 差分確認 | frmChangeConfirmation | 差分表示モーダル（Reactコンポーネント） |

※ 上記はReact Webアプリ（先行開発）での実装方針。後続でExcel VBA版UIも同等の画面を構築する。
