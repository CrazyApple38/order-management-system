2. 問題が発生するパターン
パターン1: 別の営業担当者による誤上書き
営業Aが契約先Xから「現場α 5/10 3名」の連絡を受け入力
契約先Xが営業B（現場β担当）に「現場αは5/10 2名に変更で」と伝える
営業Bがそのままシステムに入力 → 3名が2名に上書きされる
実は契約先Xの伝言が間違いで、本当は3名のままだった
営業Aが「誰がいつ変えた？」 → 変更ログで確認
元の3名にロールバックして復元
パターン2: 聞き間違いによる誤キャンセル
営業担当者が契約先から連絡を受ける
「5/10の現場αはキャンセルで」と聞き間違える
配置担当者にキャンセルを伝え、status → cancelled に変更
実際はキャンセルではなく、人数変更だった
変更履歴からキャンセル前の状態にロールバック
正しい人数に修正して保存
パターン3: 同時編集による競合 → スマートマージ
営業Aが画面を開く（version=2）
営業Bも同じ画面を開く（version=2）
営業Aが時間を変更して保存 → 成功（version=3）
営業Bが人数を変更して保存 → version不一致を検知
変更項目を比較 → 重複なし（時間 vs 人数）
マージダイアログ：両方の変更を統合して保存
重複がある場合: 両者が同じ項目（例: 人数）を変更 → 項目ごとにどちらの値を採用するか選択ダイアログを表示

4. システムによる解決策

1変更履歴の自動記録

受注情報の変更時に、変更カラム名・変更前の値・変更後の値・変更者・変更日時を自動記録。「誰がいつ何を変えたか」が即座に分かる。

2任意バージョンへのロールバック

変更履歴一覧から任意の過去バージョンを選択して復元可能。誤った上書きや誤キャンセルからの迅速な復旧を実現。

3楽観的ロック + スマートマージ

保存時にバージョン番号を照合。競合検知時、変更項目が重複しなければ両方の変更をマージ。重複する場合は項目ごとにどちらの値を採用するか選択ダイアログで確認。
